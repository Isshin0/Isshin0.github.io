HCIA 04 网络层协议及IP编址

Foreword

  • IPv4 (Internet Protocol Version 4)协议族是TCP/IP协议族中最为核心的协议族。
  • 它工作在TCP/IP协议栈的网络层，该层与OSI参考模型的网络层相对应。
  • 网络层提供了无连接数据传输服务，即网络在发送数据报文时不需要先建立连接，每一个IP数据报文独立发送。
  • 介绍IPv4地址的基本概念，介绍如何进行子网划分，并且会介绍网络IP地址规划和IP地址的基本配置。

Objectives

  • 描述网络层的主要协议
  • 描述IPv4地址的概念、分类及特殊IP地址
  • 计算IP网络以及IP子网
  • 掌握IP网络地址规划方式

IP协议

  • IP
    • Internet Protocol的缩写
    • Internet Protocol本身是一个协议文件的名称
      • 该协议文件的内容非常少，主要是定义并阐述了IP报文的格式
    • 经常被提及的IP，一般不是特指Internet Protocol这个协议文件本身
      • 而是泛指直接或间接与IP协议相关的任何内容
  1. 作用：
    1. 为网络层设备提供逻辑地址（IP地址）
    2. 负责数据包的寻址和转发
      • 网络层的PDU被称为数据包
  2. 版本:IPv4(IP Version 4)和IPv6(IP Version 6)

IPv4报文格式

IPv4-Packet-Format
  • IP Packet（IP数据包），其包头主要内容如下：
    • Version：4 bit，4：表示为IPv4；6：表示为IPv6。
    • Header Length：4 bit，首部长度，如果不带Option字段，则为20，最长为60。
    • Type of Service：8 bit，服务类型。只有在有QoS差分服务要求时，这个字段才起作用。
    • Total Length：16 bit，总长度，整个IP数据包的长度。
    • Identification：16 bit，标识，分片重组时会用到该字段。
    • Flags：3 bit，标志位。
    • Fragment Offset：12 bit，片偏移，分片重组时会用到该字段。
    • Time to Live：8 bit，生存时间。
    • Protocol：8 bit，协议：下一层协议。指出此数据包携带的数据使用何种协议，以便目的主机的IP层将数据部分上交给哪个进程处理。
      • Protocol常见值：
        • 1: ICMP, Internet Control Message；
        • 2: IGMP, Internet Group Management；
        • 6: TCP , Transmission Control Protocol；
        • 17: UDP, User Datagram Protocol。
    • Header Checksum：16 bit，首部检验和。
    • Source IP Address：32 bit，源IP地址。
    • Destination IP Address：32 bit，目的IP地址。
    • Options：可变，选项字段。
    • Padding：可变，填充字段，全填0。

  1. 报文基本信息（第一行）

    1. version：版本,v4或v6
    2. header length：头部长度，指的是整个IP报头的长度，大小为20-60字节
      • （头部长度=固定长度+可选长度）
    3. type of service：服务类型，在QoS中才有作用
    4. total length：总长度，包括IP报头和数据段（总长度=IP报头+TCP/UDP报头+数据载荷）

  2. 分片信息（第二行）

    1. 分片ID：分片标识符，当分片标识符一致则表示是同一个数据的不同分片
    2. flags：标志位，里面包含DF（0表示可以分片，1表示不可以分片）和MF（0表示最后一个分片，1表示不是最后一个分片）
    3. 分片偏移量：用于重组时定位分片原本所在的位置，保证接收设备可以正确的还原数据

  3. 路由转发信息（第三行）

    1. TTL：生存时间，数值默认为255，每经过一台三层设备该数值为减1，当设备收到TTL=0的报文时会丢弃并且回复一个ICMP错误报告给源IP设备，告知该报文因TTL=0而被丢弃
    2. protocol：协议，用于标识网络层协议或上层协议（ICMP：1；IGMP：2；TCP：6；UDP：17）
    3. header checksum：头部校验和，用于检验报文的完整性，防止数据丢失或被篡改
    4. 源IP地址：发送者的IP地址
    5. 目的IP地址：接收者的IP地址

IPv4地址

  1. 一个IPv4地址的长度为：32bit
  2. IPv4地址通常的表示方法：点分十进制
  3. 数制转换：
    1. 二进制转为十进制（8421法）
      1. 在二进制的情况下，每8bit去进行转换，表示为2的0-7次方，也就是1、2、4、8、16、32、64、128，如果该位置的二进制为1则取对应的数值，如果是0则不取。
      2. 十进制转为二进制

  1. IPv4地址范围：0.0.0.0-255.255.255.255
  2. IP地址构成：
    1. 网络部分（网络号）：用于表示一个网络（网络号相同则代表同一个网段）
    2. 主机部分（主机号）：用于区分一个网络内部的不同主机

supplement
  • IPv4地址由如下两部分组成：
    • 网络部分 (网络号)：用来标识一个网络。
      • IP地址不能反映任何有关主机位置的地理信息，只能通过网络号码字段判断出主机属于哪个网络。
      • 对于网络号相同的设备，无论实际所处的物理位置如何，它们都是处在同一个网络中。
    • 主机部分 (主机号)：用来区分一个网络内的不同主机。
  • 网络掩码 (Netmask)，又称子网掩码 (Subnet Mask):
    • 网络掩码为32 bit，与IP地址的位数一样，通常也以点分十进制数来表示。
    • 网络掩码不是一个IP地址，在二进制的表示上是一堆连续的1、后面接一堆连续的0。
    • 通常将网络掩码中1的个数称为这个网络掩码的长度。
    • 如：掩码0.0.0.0的长度是0，掩码252.0.0.0的长度是6。
    • 网络掩码一般与IP地址结合使用，其中值为1的比特对应IP地址中的网络位；值为0的比特对应IP地址中的主机位，以此来辅助我们识别一个IP地址中的网络位与主机位。
    • 即网络掩码中1的个数就是IP地址的网络号的位数，0的个数就是IP地址的主机号的位数。

  1. 子网掩码：
    1. 作用：用于区分IP地址中的网络部分和主机部分
    2. 构成：由连续的1和连续的0构成，其中连续的1代表网络部分
    3. 书写方式：点分十进制、以连续的1的位数表示
  2. 网络寻址
    1. 二层寻址：同个网段内的通信，直接通过IP地址寻找目的设备
    2. 三层寻址：不同网段的通信，需要利用网关设备转发不同网络的报文
supplement
  • 网络号用于表示主机所在的网络，类似于“XX省XX市XX区XX小区”的作用。
  • 主机号用于表示网络号所定义的网络范围内某个特定的主机接口，类似于门牌号“XX栋XX号”的作用。
  • 网络寻址：
    • 二层网络寻址：可直接通过IP地址，找到对应的主机接口。
    • 三层网络寻址：利用网关转发来自不同网段之间的数据包。
  • 网关：
    • 报文转发过程中，首先需要确定转发路径以及通往目的网段的接口。如果目的主机与源主机不在同一网段，报文需要先转发到网关，然后通过网关将报文转发到目的网段。
    • 网关是指接收并处理本地网段主机发送的报文并转发到目的网段的设备。为实现此功能，网关必须知道目的网段的IP地址。网关设备上连接本地网段的接口地址即为该网段的网关地址。

  1. IP地址分类（有类编址)：
    • A类：前1bit固定为0，默认掩码为8位，范围为0.0.0.0-127.255.255.255，用于主机
    • B类：前2bit固定为10，默认掩码为16位，范围为128.0.0.0-191.255.255.255，用于主机
    • C类：前3bit固定为110，默认掩码为24位，范围为192.0.0.0-223.255.255.255，用于主机
    • D类：前4bit固定为1110，范围为224.0.0.0-239.255.255.255，用于组播
    • E类：前4bit固定为1111，范围为240.0.0.0-255.255.255.255，用于科研
    • 通过是被IP地址第一个十进制数在哪一个范围就可以确认该IP地址属于哪一类IP地址。

  1. IP地址类型：
    1. 网络地址：用于定义一个网络，该地址不可分配给主机使用（网络位不变，主机为全为0）
    2. 广播地址：用于给一个网络中所有主机发送数据，该地址不可分配给主机使用（网络位不变，主机位全为1）
    3. 可用地址：网络中除了网络地址和广播地址之外的其他地址，可以分配给主机使用
      （一个网络中的可用地址数：2^N-2，N为该网络的主机位）

  1. 公网地址和私网地址
    1. 公网地址由IANA组织进行分配，国内用户获取公网IP地址需要向运营商购买，公网IP地址全网唯一
    2. 私网地址：用于局域网内部，不同局域网之间的私网地址可以重复，在A\B\C三类地址中各自划分出一个范围作为私网IP地址。
      • A类：10.0.0.0-10.255.255.255
      • B类：172.16.0.0-172.31.255.255
      • C类：192.168.0.0-192.168.255.255
  2. 特殊地址
    1. 有限广播地址：255.255.255.255，所有设备都会监听该地址
    2. 任意地址：0.0.0.0，代表所有IP地址
    3. 环回地址：127.0.0.0/8，用于测试设备本身的软件系统
    4. 本地链路地址：169.254.0.0/24，当主机进行DHCP自动获取IP地址失败之后，就会使用本地链路地址作为临时地址

子网划分

  • 使用可变长子网掩码（无类编址），根据实际需求进行子网划分，在原本IP的基础上，网络位向主机位借位。
    1. 一个IP网段可以划分的子网数量：2^m，m为借位数
    2. 一个IP网段的子网的可用IP地址数量：2^n-2（n为主机位数）
    3. 子网的网络地址：子网的主机位全为0
    4. 子网的广播地址：子网的主机位全为1

ICMP协议

  1. 定义：IP协议的辅助协议，用于在网络设备中传递各种差错和控制信息。

    • ICMP协议用来在网络设备间传递各种差错和控制信息，对于收集各种网络信息、诊断和排除各种网络故障等方面起着至关重要的作用
    • ICMP消息封装在IP报文中，IP报文头部Protocol值为1时表示ICMP协议
      • ICMP消息的格式取决于Type和Code字段，其中Type字段为消息类型，Code字段包含该消息类型的具体参数

  2. 报文格式：

    1. echo reply和request用于ping功能
      1. type 0 ，Echo reply
      2. type 8 ，Echo request
    2. type 3代表不可达类型（用于tracert功能）
    3. type 5用于重定向功能
    4. type 11用于TTL超时

  3. 作用：

    1. 控制：ICMP重定向
      • 当网络中某台设备发现接收报文和发送报文的端口是同一个时，就会触发ICMP重定向功能，发送出ICMP重定向报文，让访问的设备下一次直接走最优的路径。
      • ICMP重定向功能默认关闭，因为安全性较低，可以手动开启。
    2. ICMP差错检测：ping功能
      1. 作用：用于监测源目设备之间的网络连通性
      2. 常用选项：
        1. -a：指定源IP
        2. -t：一直不间断发送数据包
        3. -h：指定TTL值，默认255
    3. ICMP错误报告：tracert功能
      • 通过不同的类型和代码的报文得到不同的错误报告，用于监测网络的故障点以及检测网络的路径。
Ping
  • Ping是检测网络连通性的常用工具，同时也能够收集其他相关信息。
  • 用户可以在Ping命令中指定不同参数，如ICMP报文长度、发送的ICMP报文个数、等待回复响应的超时时间等
  • 设备根据配置的参数来构造并发送ICMP报文，进行Ping测试

tracert的具体过程
  • Tracert基于报文头中的TTL值来逐跳跟踪报文的转发路径。
  • 为了跟踪到达某特定目的地址的路径，源端首先将报文的TTL值设置为1。
  • 该报文到达第一个节点后，TTL超时，于是该节点向源端发送TTL超时消息，消息中携带时间戳。然后源端将报文的TTL值设置为2，报文到达第二个节点后超时，该节点同样返回TTL超时消息，以此类推，直到报文到达目的地。
  • 这样，源端根据返回的报文中的信息可以跟踪到报文经过的每一个节点，并根据时间戳信息计算往返时间。

网络IP地址规划

  • IP地址规划要和网络结构、路由协议、流量规划、业务规则等结合起来考虑。
  • IP地址的规划应尽可能和网络层次相对应，应该是自顶向下的一种规划
  • 总而言之，IP地址规划的目标是：易管理，易扩展，利用率高
规划原则
  • 唯一性：一个IP网络中不能有两个主机采用相同的IP地址。
  • 连续性：连续地址在层次结构网络中易于进行路由汇总，大大缩减路由表，提高路由计算的效率、加速路由收敛。
  • 扩展性：地址分配在每一层次上都要有合理的预留，在网络规模扩展时能保证路由汇总所需的连续性。避免网络扩展造成的地址、路由重新规划。
  • 结构化、业务相关性：地址规划与网络拓扑结构和网络承载业务结合起来，便于路由规划和QoS部署。好的IP地址规划使得每个地址都具有实际含义，看到一个地址就可以大致判断出该地址所属的设备和对应的业务。

Summary

  • 只要在IP网络上，用户想要连接到Internet，就需要IP地址
